# EvotorEReceiptApp

Минимальное Android Java-приложение для смарт-терминала Эвотор 7.3, которое формирует и отправляет электронный чек на email и/или телефон.

## Что делает
- принимает из формы:
  - наименование товара
  - количество
  - цену
  - email покупателя
  - телефон покупателя
  - paymentPlace (адрес сайта/место расчёта)
- создаёт чек продажи
- отправляет его через `PrintSellReceiptCommand`
- показывает расширенную ошибку ККТ, если SDK её вернул

## Важно
Официальная документация требует:
- permission `ru.evotor.permission.receipt.print.INTERNET_RECEIPT`
- указать `clientEmail` и/или `clientPhone`
- для интернет-чека заполнить `paymentPlace`

## Подключение SDK Эвотор
В этот архив я **не включал** бинарные библиотеки SDK Эвотор. Положите их в папку:

`app/libs/`

В `app/build.gradle` уже стоит:

```gradle
implementation fileTree(dir: 'libs', include: ['*.jar', '*.aar'])
```

Если у вас используется другой способ подключения SDK, замените этот блок на свой.

## Возможная разница версий SDK
В документации Эвотор и в демо-примерах встречаются разные overload/конструкторы `PrintSellReceiptCommand` и `PrintGroup`.

В `MainActivity.java` я оставил:
- основной вызов по официальной схеме из документации,
- и комментарий с альтернативой, если ваша локальная версия SDK ожидает другой конструктор.

## Сборка
Открыть проект в Android Studio, подключить SDK Эвотор в `app/libs`, затем собрать APK.
